# 更新记录

## 0.4.1

- 支持通过 `CONTEXT-MAP.md` 定位多业务域术语表、决定记录和 ADR，保持系统级与业务域上下文边界。
- 扩展跨会话交接，增加成果引用、当前状态、验证边界、敏感信息处理和下一 Skill 建议。
- 收紧 `jq-pm-grilling`、`jq-pm-review`、`jq-pm-prototype` 与 `jq-ui-spec` 的自动触发边界。
- 为 `jq-pm-research` 增加可选并行 Agent 资料收集，并由主 Agent 统一复核和综合。
- 将关键行为改为正向目标表达，保留真实性、安全、权限和数据边界等必要护栏。
- 新增多业务域、并行调研、跨会话交接和 UI 专项审查验收题。

## 0.4.0

- 新增第一性原理产品推导协议，明确触发条件、事实与约束分类、最小因果链、推导卡、验证和推翻条件。
- 将第一性原理作为 8 个 Skill 共用的决策推导层；深度推导由 `jq-pm-grilling` 承担，其他 Skill 执行阶段化快速检查。
- 增强产品上下文协议，让高影响决定记录目标结果、事实、硬约束、已移除惯例和推翻条件。
- 增加跨平台第一性原理验收题，并为发现、调研、PRD、原型、评审和汇报补充对应行为要求。
- 扩展发布前自检，确保全部 8 个 Skill 已接入第一性原理协议。

- 重写 README 安装部分，支持将安装提示词和 GitHub 地址直接交给对应 AI。
