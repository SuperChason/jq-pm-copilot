# 企业产品汇报

从现有材料提炼统一口径，按接收对象决定内容密度和表达方式。

## 能力说明

- **核心任务**：从项目材料中提取当前有效口径，形成适合具体受众、渠道和沟通目的的内容。
- **最小输入**：要传达的事情和目标受众；优先补充主版本材料、最新进展、沟通目的、篇幅和期望动作。
- **主要产出**：可直接发送的短消息、领导汇报、评审发言、演示串词、周报月报、风险说明和决策请求。
- **表达原则**：结论和影响优先，事实准确，行动明确；同一事实在不同受众版本中保持一致。
- **能力边界**：不虚构成果和数据，不用过期材料覆盖当前口径，不代替正式 PRD、项目计划或决策记录。
- **上游来源**：其他 7 个 Skill 的阶段产物、产品上下文、会议纪要、进展记录、数据和已确认决策。
- **下游衔接**：材料存在逻辑缺口时回到对应 Skill 修正；重大判断缺少证据时进入 jq-pm-research。

## 工作流程

1. 明确接收对象、发送场景、期望动作、篇幅和截止时间。
2. 按 ../jq-pm-grilling/references/product-context-protocol.md 读取项目主版本、当前决定、假设和术语，提取事实、数据、风险和下一步。
3. 删除过程噪声，把内容组织为结论、依据、影响和行动。
4. 按对象选择结构，并使用 references/briefing-style-guide.md 调整口径。
5. 对未经确认的数字和结论做显式标记，不补造漂亮数据。
6. 直接交付可复制版本；有价值时再附一个更短版本。

## 对象适配

- 领导：先结论和业务价值，再讲关键进展、风险和需要决策的事项。
- 研发与测试：讲清版本范围、规则变化、依赖、验收和时间点。
- 销售与客户：突出场景价值、能力边界、使用条件和可承诺范围。
- 项目群：说明发生了什么、影响谁、谁在何时做什么。
- 正式汇报：形成背景、目标、成果、问题、计划和决策请求的完整叙事。

## 默认输出

1. 标题或开场句
2. 可直接发送的正文
3. 需要时附“再短一点”版本
4. 仅在必要时附口径依据与待确认信息

表达要自然、具体、克制。避免空泛口号、流水账和无法落地的“持续推进”。

交付前检查：受众在第一次阅读后能说清“发生了什么、为什么重要、对我有什么影响、接下来谁做什么”；数字、日期、版本和责任人都与主材料一致；正文可以直接复制使用。
