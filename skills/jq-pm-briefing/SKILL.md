# 企业产品汇报

从现有材料提炼统一口径，按接收对象决定内容密度和表达方式。

## 工作流程

1. 明确接收对象、发送场景、期望动作、篇幅和截止时间。
2. 读取项目主版本材料，提取事实、数据、决定、风险和下一步。
3. 删除过程噪声，把内容组织为结论、依据、影响和行动。
4. 按对象选择结构，并使用 references/briefing-style-guide.md 调整口径。
5. 对未经确认的数字和结论做显式标记，不补造漂亮数据。
6. 直接交付可复制版本；有价值时再附一个更短版本。

## 对象适配

- 领导：先结论和业务价值，再讲关键进展、风险和需要决策的事项。
- 研发与测试：讲清版本范围、规则变化、依赖、验收和时间点。
- 销售与客户：突出场景价值、能力边界、使用条件和可承诺范围。
- 项目群：说明发生了什么、影响谁、谁在何时做什么。
- 正式汇报：形成背景、目标、成果、问题、计划和决策请求的完整叙事。

## 默认输出

1. 标题或开场句
2. 可直接发送的正文
3. 需要时附“再短一点”版本
4. 仅在必要时附口径依据与待确认信息

表达要自然、具体、克制。避免空泛口号、流水账和无法落地的“持续推进”。
