# 企业 UI 规范

为企业后台产品提供可执行的设计基线。既可以指导生成界面，也可以检查已有页面。

## 选择任务

- 生成或改造界面：依次读取 references/design-decisions.md、design-tokens.md、components.md、page-patterns.md、interaction-states.md 和 accessibility.md。
- 审查界面：读取上述文件及 references/review-checklist.md；若有代码，运行 scripts/check-enterprise-ui.mjs。
- 制作原型：结合 jq-pm-prototype 使用 assets/prototype-starter。

## 生成流程

1. 读取项目现有设计系统、品牌约束和技术栈。项目已有明确规范时优先遵循，并记录与本基线的差异。
2. 识别页面类型、用户核心任务、信息层级和必需状态。
3. 使用设计令牌，避免在页面中散落新的颜色、圆角、阴影和间距值。
4. 先组合标准组件和页面模式，再处理业务特有结构。
5. 补齐加载、空、错误、无权限、禁用、成功和部分完成状态。
6. 检查键盘操作、焦点、对比度、标签和错误提示。
7. 在浏览器中检查常见桌面视口和关键交互。

## 审查流程

1. 收集页面截图、原型、代码和对应 PRD。
2. 先判断任务闭环和信息层级，再查视觉细节。
3. 按 review-checklist.md 分为阻断、重要、改进和建议。
4. 每条意见包含位置、当前表现、规范依据、建议值和验证方式。
5. 能自动修复且用户要求修改时，只改与本次任务相关的内容。

## 强制基线

- 默认主题色为 #3E5BF2，正文主色为 #1A1A1A，页面背景为 #F5F5F7。
- 默认组件圆角为 4px；胶囊标签和圆形控件属于明确例外。
- 侧栏选中态使用主题色背景和白色文字。
- 正文基础字号 14px，常规行高 1.5。
- 交互元素必须提供清晰焦点状态，不能只依赖颜色表达状态。
- 页面中的业务规则来自 PRD 或显式假设，UI 规范不能代替业务决策。

本 Skill 基于团队提供的 原始企业 UI 规范 v1.2 整理。冲突项和本阶段取值记录在 references/design-decisions.md。
