# 无障碍与可用性基线

- 使用语义化 HTML；页面只有一个主标题，标题层级连续。
- 所有输入控件有可见标签；占位符不能代替标签。
- 键盘可以完成主流程，Tab 顺序与视觉顺序一致。
- 自定义控件具有正确角色、名称、状态和键盘行为。
- 图标按钮提供可访问名称；装饰图标对辅助技术隐藏。
- 正文与背景的对比度至少 4.5:1，大号文字至少 3:1。
- 焦点状态始终可见，弹窗打开后聚焦首个合理位置，关闭后回到触发控件。
- 错误信息与对应字段关联，同时说明修正方式。
- 状态和图表不能只依赖颜色，增加文字、图标、线型或形状。
- 页面缩放到 200% 时无横向关键内容丢失，核心操作仍可完成。
- 表格包含清晰表头；复杂表格说明单位、排序和筛选状态。
- 动效遵循 prefers-reduced-motion。
