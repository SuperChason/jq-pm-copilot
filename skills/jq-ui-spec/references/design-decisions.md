# UI 规范设计决策

来源：团队提供的 原始企业 UI 规范 v1.2。第一阶段对原文件中的冲突和缺口采用以下统一口径。

## 已统一

| 主题 | 第一阶段口径 | 理由 |
| --- | --- | --- |
| 品牌主色 | #3E5BF2 | 原规范主色和完整 CSS 变量均采用该值 |
| 圆角 | 小 2px、中 4px、大 6px | 与原规范尺寸表和组件示例一致 |
| 默认圆角 | 4px | 用于中尺寸按钮、输入框、页面卡片 |
| 侧栏激活文字 | 白色 | 保证主色侧栏上的对比度和一致性 |
| 侧栏激活变量 | --sidebar-active-bg | 替换页面模板中未定义的 --sidebar-active |
| 桌面基线 | 1440px 设计板，兼容 1280px | 保留企业后台 PC 优先定位，同时要求常见视口可用 |
| 像素要求 | 规范值作为组件基线 | 页面可按信息密度选择已定义的大中小尺寸 |

## 新增质量门槛

- 所有交互控件有键盘焦点状态。
- 状态不能只用颜色表达，需同时使用文字、图标或形状。
- 表单错误与字段关联，并提供可执行的修正信息。
- 关键页面覆盖加载、空、错误、无权限和成功状态。
- 页面缩放到 200% 时核心任务仍可完成。

## 待公司确认

- 公司正式中英文字体及授权文件。
- Logo、安全区、产品命名和品牌资产使用规则。
- 移动端、数据大屏和门户站点是否另设规范。
- 与现有前端组件库的映射关系和版本治理责任人。
