# 交互与页面状态

## 控件状态

所有可交互控件至少覆盖默认、hover、focus-visible、active 和 disabled。输入控件另需只读、已填写、校验中、错误和成功状态。

焦点环建议使用 2px 主色外框并留 2px 间距。禁用控件不可响应点击；只读控件允许复制并保持可识别。

## 页面状态

- 加载：首屏使用骨架或局部加载，避免整页长时间空白。
- 空：说明当前状态、可能原因和下一步动作。
- 错误：解释影响、保留用户已输入内容，并提供重试或恢复路径。
- 无权限：说明权限名称、申请人或联系路径。
- 成功：说明已完成什么、状态何时生效和后续入口。
- 部分完成：列出成功、失败和可重试项，避免只给“操作失败”。

## 反馈时机

即时校验在用户完成字段输入后触发，避免每次按键都报错。短操作用消息提示；需要处理的结果用页面内提示或结果页。不可逆操作明确影响对象和数量，并二次确认。

## 动效

动效只用于解释状态和层级变化。常规反馈控制在 120 至 240ms，并尊重系统减少动态效果设置。
