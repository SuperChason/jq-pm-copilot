# UI 审查清单

## 任务与内容

- 页面主任务和主操作清楚。
- 信息层级与用户决策顺序一致。
- 术语、字段、状态和 PRD 保持一致。
- 示例数据具有真实业务语义。

## 布局与视觉

- 侧栏、顶栏、内容区和页面卡片使用统一框架。
- 颜色、字号、间距、圆角、阴影来自设计令牌。
- 同类组件尺寸一致，主次操作层级明确。
- 1280×800 和 1440×900 下核心操作可见可用。

## 组件与状态

- 表格列宽、对齐、行高、操作列和分页合理。
- 表单标签、必填、帮助、校验和错误完整。
- 加载、空、错误、无权限、禁用、成功和部分完成状态齐全。
- 弹窗、抽屉和不可逆操作具有完整关闭、取消和确认路径。

## 无障碍

- 键盘可达、焦点可见、控件名称明确。
- 文本与背景对比度符合要求。
- 状态不只依赖颜色。
- 缩放、长文本和辅助技术使用不阻断主流程。

## 意见严重度

阻断：主任务无法完成或存在高风险误操作。重要：规则、状态或一致性会造成明显误解。改进：效率、可读性和维护性问题。建议：低风险优化。
