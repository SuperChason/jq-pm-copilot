# 产品上下文协议

产品上下文用于让发现、研究、PRD、原型、评审和汇报共享同一组事实、术语、决定和假设。优先复用项目已有文档与目录，不为了统一格式迁移成熟项目。

## 进入项目时

1. 先查找项目索引、主版本 PRD、发现简报、术语表、决定记录和假设清单。
2. 根目录存在 `CONTEXT-MAP.md` 时，先确认当前任务所属业务域以及系统级影响，再读取对应业务域的术语表和决定记录。
3. 确认每份材料的版本、日期、状态和适用范围。
4. 发生冲突时按“用户本轮明确确认 > 已接受决定记录 > 当前主版本文档 > 历史材料”的顺序处理。
5. 无法消除的冲突保持显式，并记录需要谁确认。

## 默认载体

项目没有既有约定，并且当前任务已获准创建或更新文档时，按需使用：

| 信息 | 默认载体 | 内容边界 |
| --- | --- | --- |
| 产品发现结论 | PRODUCT-BRIEF.md 或项目现有发现简报 | 用户、场景、问题、目标、范围、风险 |
| 规范术语 | CONTEXT.md，或 CONTEXT-MAP.md 指向的业务域 CONTEXT.md | 仅记录领域术语和定义，不写实现细节 |
| 工作决定 | docs/product/decision-log.md | 日期、状态、决定、理由、影响、来源；高影响决定附第一性推导卡 |
| 工作假设 | docs/product/assumptions.md | 假设、错误代价、验证动作、责任人和最晚确认点 |
| 长期决定 | docs/adr/NNNN-slug.md | 仅记录难逆转、缺少背景会令人意外且经过真实取舍的决定 |

按需创建文件。第一个术语确认后再创建 CONTEXT.md，第一个工作决定或假设形成后再创建对应文件，第一个满足条件的长期决定形成后再创建 docs/adr。

## 多业务域上下文

项目存在多个能够独立演进的业务域，并且根目录已有 `CONTEXT-MAP.md` 时，按映射维护上下文：

- `CONTEXT-MAP.md` 至少记录业务域、适用范围、术语表路径、决定记录或 ADR 路径和建议维护人。
- 系统级术语与跨业务域决定保留在根级载体；业务域特有术语和决定进入对应业务域载体。
- 同一概念只保留一个规范定义；其他业务域通过引用复用，并明确局部差异。
- 当前任务同时影响多个业务域时，分别记录影响并指出需要共同确认的接口、状态或责任边界。

单一业务域项目继续使用一个 CONTEXT.md。只有业务域已经具备独立语言和演进边界时才建立映射，保持上下文结构与项目复杂度匹配。

## 术语记录

CONTEXT.md 只维护项目特有的领域语言。推荐格式：

    **规范术语**：
    一至两句定义，说明它与相邻概念的边界。
    _避免使用_：历史叫法、容易混淆的别名

同一概念只保留一个规范名称。修改术语时列出受影响的 PRD、原型、接口、数据字段和测试用例。

## 工作决定

每条决定至少包含：

- 日期与状态：提议、已确认、已废弃或被替代。
- 决定内容：用一句可执行的话表达。
- 选择理由：说明证据、约束和关键取舍。
- 影响范围：用户、流程、规则、页面、数据、权限、接口或里程碑。
- 来源：会议、用户确认、调研证据或正式评审。
- 推导依据：触发第一性原理时，记录目标结果、已验证事实、硬约束、已移除惯例、最小因果链、最小验证和推翻条件。

普通产品决定进入 decision-log。只有同时满足以下条件才创建 ADR：

1. 以后改变的成本明显。
2. 缺少背景时，未来读者会质疑这个选择。
3. 确实比较过有意义的替代方案。

ADR 默认只写标题以及一至三句话的背景、决定和理由。按 docs/adr 中现有最大编号递增。

## 工作假设

每条假设至少包含：

- 暂定答案及其适用范围。
- 假设错误时的产品或交付影响。
- 最小验证动作和判定标准。
- 建议责任人。
- 最晚确认点。
- 当前状态：待验证、已验证、被否定或已转为决定。

假设被确认后转入决定记录；被否定后保留结论和影响，避免重复验证。

    - 未决问题：
    - 已执行验证：
    - 未执行验证及原因：
    - 建议下一 Skill：

下游 Skill 先读取交接块，再按需追溯源文档。下游发现冲突时停止传播冲突口径，回到决定来源确认。

用户明确要求切换会话、任务或 Agent 时，将交接块扩展为跨会话交接：

- 优先引用已有 PRD、原型、评审、决定、提交和链接，避免复制已经落盘的完整内容。
- 记录当前工作分支、未提交变更、验证状态和下一步首个动作；没有对应内容时省略。
- 移除账号、密码、密钥、客户数据和可识别个人信息；敏感事实只记录安全存放位置和最小使用说明。
- 列出建议调用的 Skill 及用途，让接收方能够直接开始下一步。
- 平台支持临时文件时保存到操作系统临时目录；缺少文件写入能力时返回可直接复制的完整 Markdown。项目目录仅在用户明确要求时保存交接文件。
