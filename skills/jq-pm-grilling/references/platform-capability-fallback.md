# 跨平台能力降级协议

在 Codex、Claude Code、WorkBuddy 或其他兼容 Agent Skills 的环境中执行时，先识别当前可用的文件、联网、浏览器和脚本能力。能从环境直接判断时继续执行，不为工具名称向用户增加无效确认。

## 降级规则

- 无文件读取能力：请用户提供关键材料，或基于当前输入交付完整成稿；不得声称已经读取项目文件。
- 无文件写入能力：在回复中返回可直接保存的完整内容，并给出建议文件名；不得声称已经落盘。
- 无联网能力：继续处理已有材料，把需要时效性证据的结论标为待核验；不得编造来源、链接或最新事实。
- 无浏览器或截图能力：改为执行页面结构、交互状态和代码静态走查，附人工浏览器验收清单；不得声称已经完成视觉实测。
- 无脚本运行能力：按同一检查标准人工核对，列出未执行的自动检查及其影响；不得把静态判断包装成脚本结果。
- 无法读取相邻 Skill：先完成当前 Skill 可以独立完成的部分，输出标准下一阶段交接，并明确建议安装完整套件；不得声称已经调用缺失的 Skill。
- 无长期上下文能力：在交付末尾附产品上下文交接块，至少包含事实、决定、假设、术语、未决问题和下一步。
- 权限被拒绝：说明受阻能力、对交付的实际影响和所需最小权限；能降级时继续完成其余工作。

## 交付说明

收尾时区分已经实际完成的动作、采用的降级方式和仍需人工完成的验证。只报告真实执行结果。
