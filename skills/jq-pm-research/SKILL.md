# 企业产品研究

先明确调研要支持的决定，再收集证据，最终回答“产品应该做什么”。

执行前读取 ../jq-pm-grilling/references/platform-capability-fallback.md。缺少联网或文件能力时降低结论确定性，保留待核验项，不虚构来源或已执行动作。

## 能力说明

- **核心任务**：围绕一个具体产品决策，收集、核验和组织用户、竞品、行业、政策或技术证据。
- **最小输入**：一个需要判断的问题；优先补充决策背景、研究对象、时间地域范围、比较维度和交付期限。
- **主要产出**：结论摘要、证据表、对象对比、趋势判断、产品启发、建议动作和待验证事项。
- **证据原则**：优先官方和一手来源；记录发布日期、事件日期和适用范围；区分事实、推断和建议。
- **能力边界**：不以材料数量代替结论质量，不用无法核验的转述支撑重大判断，不自动把研究启发升级为正式需求。
- **上游来源**：jq-pm-discovery 的信息缺口、jq-pm-grilling 的待验证假设、路线规划问题、竞品观察任务、方案选型或事实核查请求。
- **下游衔接**：确认后的结论进入 jq-pm-prd；需要对外说明时进入 jq-pm-briefing；研究问题仍含糊时退回 jq-pm-discovery。

## 工作流程

1. 明确决策问题、使用者、时间范围、地域范围和交付形式。问题建立在竞品惯例、行业共识或未经验证的因果关系上时，读取 ../jq-pm-grilling/references/first-principles-protocol.md，把需要核验的底层事实和关键因果假设单独列出。
2. 按 ../jq-pm-grilling/references/product-context-protocol.md 检查项目内已有资料、决定和假设，复用可信结论并标出日期。
3. 制定最小调研计划：必须回答的问题、优先来源和停止条件。先验证决定成立所必需的事实，再比较竞品功能和行业做法。
4. 平台支持并行 Agent，并且调研可以拆成互相独立的来源或对象时，可并行完成资料收集；主 Agent 保留决策问题、来源标准、冲突核验和最终综合。平台缺少并行能力时按相同标准顺序完成。
5. 涉及当前事实、具体网页或高风险结论时联网核验；优先官方和一手来源。
6. 记录来源、发布日期、事件日期和证据强度，区分事实、分析与推断。
7. 对冲突信息做交叉核验，说明无法消除的不确定性。
8. 将发现映射到用户价值、产品能力、差异化、风险和优先级。
9. 输出可执行建议、仍需验证的关键假设和下一阶段交接。

## 质量要求

- 每条重要事实都有可追溯来源。
- 时效性强的资料标注核验日期。
- 竞品宣传、媒体转述和用户评论分别注明证据属性。
- 调研范围以支持当前决策为边界，避免材料堆积。
- 建议必须能回溯到证据或明确标注的推断。
- 明确区分跨场景仍成立的底层事实、特定市场做法、单个组织约束和纯粹类比。
- 并行 Agent 负责资料收集时，主 Agent 逐条复核进入最终结论的关键证据并统一引用口径。

## 输出格式

1. 结论摘要
2. 本次要支持的产品决策
3. 调研范围与方法
4. 关键发现及证据
5. 对比表或主题归纳
6. 对产品规划的启发
7. 建议行动、优先级与理由
8. 风险、局限与待验证项
9. 来源清单

开始前读取 references/research-source-policy.md。若研究问题本身含糊，先调用 jq-pm-discovery。

交付前检查：每条关键结论都能回到具体证据；读者能看出哪些是事实、哪些是分析；建议动作明确说明支持什么决策、优先级和验证方式。无法满足时，补充来源或降低结论确定性。
